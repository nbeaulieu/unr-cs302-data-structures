# Pseudocode Examples

This directory contains pseudocode implementations of algorithms used in CS 302. These are designed for teaching concepts before diving into actual code implementation.

## Available Pseudocode

### Sorting Algorithms

- **`selection_sort_pseudocode.txt`** - Selection Sort (Maximum-to-End variant)
- **`bubble_sort_pseudocode.txt`** - Bubble Sort with Early Termination
- **`insertion_sort_pseudocode.txt`** - Insertion Sort (Shifts, not swaps)

## Format

Each pseudocode file includes:
- **Algorithm Overview**: High-level description and strategy
- **Basic Algorithm**: Complete pseudocode implementation
- **Step-by-Step Example**: Detailed walkthrough with example data
- **Key Teaching Points**: Important concepts and common mistakes
- **Complexity Analysis**: Time and space complexity breakdown
- **When to Use**: Practical guidance on algorithm selection

## Using Pseudocode in Teaching

1. **Introduce the Concept**: Use pseudocode to explain the algorithm strategy
2. **Walk Through Examples**: Follow the step-by-step examples together
3. **Discuss Complexity**: Analyze time/space complexity from pseudocode
4. **Translate to Code**: Convert pseudocode to actual C++ implementation
5. **Compare Algorithms**: Use pseudocode to compare different approaches

## Pseudocode Notation

- `←` : Assignment (e.g., `x ← 5`)
- `IF...THEN...END IF` : Conditional statements
- `FOR...DO...END FOR` : Loops
- `WHILE...DO...END WHILE` : While loops
- `SWAP(a, b)` : Exchange two values
- `RETURN` : Exit function
- `BREAK` : Exit loop early
- Comments use `//` or are in separate sections

## Benefits of Pseudocode

- **Language Independent**: Focus on algorithm, not syntax
- **Clear Logic**: Easy to see the flow and structure
- **Teaching Tool**: Helps students understand before coding
- **Documentation**: Serves as algorithm specification
- **Comparison**: Easy to compare different algorithms side-by-side
